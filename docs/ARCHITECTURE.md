# 架构说明

## 目标与边界

本项目解决一次性、大批量、保留目录层级的飞书云盘归档。它不建立长期同步连接，也不绕过账号权限：可导出的内容使用当前登录会话下载；无权限内容记录后停止重试；Base 当前跳过。

## 数据流

1. `<源目录>` 保存 `.url` 云文档链接和普通实体文件。
2. `build-feishu-task-lists.mjs` 递归读取 `.url`，解析文档类型和相对目录，生成全目录及逐文件夹 TSV 清单。
3. Chrome 扩展读取清单并复用已登录飞书页面：
   - 调用飞书网页使用的内部导出接口处理 Word、Markdown、Sheet 和 Slides；
   - 读取 MindNotes 页面已渲染的节点树，在扩展内生成 FreeMind XML；
   - 使用 Chrome Downloads API 将文件写入对应相对目录。
4. 扩展在 `chrome.storage.local` 中保存成功、无权限和失败状态，并生成 TSV 明细、运行概览及 HTML 简报。
5. `merge-feishu-archive.mjs` 把源目录中的实体文件与云端导出复制到新归档目录，使用 SHA-256 判断重复内容；同路径但内容不同的文件重命名后同时保留。

```text
本地源目录 (.url + 实体文件)
          │
          ├─ build-feishu-task-lists.mjs ─→ TSV 任务清单
          │                                  │
          │                         已登录 Chrome + 扩展
          │                                  │
          │                            云文档导出物
          │                                  │
          └──────── merge-feishu-archive.mjs ┘
                             │
                      保留层级的完整归档
```

## 格式映射

| 飞书类型 | 本地格式 | 实现方式 |
|---|---|---|
| `docx` / `docs` | `.docx` | 飞书网页内部导出接口 |
| `docx` / `docs` | `.md` 或含附件压缩包 | Markdown 导出，内容范围为所有内容 |
| `sheets` | `.xlsx` | 飞书网页内部导出接口 |
| `slides` | `.pptx` | 飞书网页内部导出接口 |
| `mindnotes` | `.mm` | 从 `.mindnote-tree` 递归提取节点并生成 FreeMind XML |
| `base` | 无 | 固定跳过并写入报告 |

格式配置的代码来源是 `chrome-extension/feishu-bulk-export/core.mjs`。

## 为什么采用浏览器扩展

公开 OpenAPI 对云盘中不同类型对象的批量、跨格式导出覆盖有限，而普通爬虫直接复用登录态通常需要处理 Cookie、CSRF、动态签名和浏览器下载行为。扩展运行在用户已经登录的 Chrome 中，可以通过网页的正常权限模型发起请求，并用 Downloads API 管理落盘路径；同时无需导出或持久化登录凭证。

代价是内部接口和页面 DOM 随飞书版本变化，需要维护。项目因此把格式参数集中在 `core.mjs`，并保留小批量验证、详细失败台账和断点续跑能力。

## 会话授权、限速与并发

后台脚本从用户正常导出请求中捕获 CSRF 和必要业务请求头，保存在 Chrome session storage；Chrome 退出后失效。扩展不读取 Cookie。创建导出任务默认限速，普通临时错误采用指数退避重试；FreeMind 固定单任务运行。

## 断点与状态

- 成功断点按相对目录、飞书类型、token 和目标扩展名区分；
- `code 1002 / no permission` 保存为终止断点，后续运行直接跳过；
- 其他失败保留在任务台账中，可以在下一轮重试；
- Base 使用“本期跳过”状态；
- HTML 简报反映 Chrome 本地台账，不代替最终文件系统核对。

## MindNotes 导出

飞书原生“下载为 → FreeMind”在部分页面版本中会点击成功但不登记下载，甚至抛出前端异常。当前实现不依赖该菜单，也不做任意 Blob 拦截，而是在临时打开的同域页面中读取 `.mindnote-tree` 的节点父子结构，转义为标准 FreeMind XML 后下载 `.mm`。

## 合并语义

合并脚本只复制、不移动输入文件，也不删除输出中的旧文件：

- `.url` 不进入归档；
- `_未下载记录`、`.tsv` 和 `.crdownload` 不进入归档；
- 目标不存在时直接复制并保留时间戳；
- 目标存在且 SHA-256 相同则跳过；
- 目标存在但内容不同则增加来源后缀，保留两个版本。

因此，干净交付必须使用新的空输出目录。归档完成后以 `归档合并汇总.json` 和实际文件计数共同验收。

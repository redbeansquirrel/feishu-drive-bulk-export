# 运行与排障

## 推荐运行顺序

1. 备份源目录，不在原目录中做批量改名或删除。
2. 生成清单，检查 `清单汇总.json` 的任务数和类型分布。
3. 重新加载扩展，手动导出一个同类型文档以初始化临时请求头。
4. 每种格式先运行 3 个任务，抽查文件能否打开、内容范围和目录是否正确。
5. 全量运行，并保持执行用的飞书标签页和 Chrome 会话有效。
6. 导出未下载明细与 HTML 简报，处理可重试错误。
7. 使用新的空目录合并归档，核对 `归档合并汇总.json`。

## 常见状态

| 状态或日志 | 含义 | 处理方式 |
|---|---|---|
| `code 1002 / no permission` | 当前账号没有导出权限 | 终止断点；记录后不重试 |
| 页面没有返回文件 token | 页面类型、跳转或加载状态不符合预期 | 打开源链接确认对象类型和访问状态，再重试 |
| 创建导出失败或轮询超时 | 临时网络、限流、会话或接口变化 | 先用单个同类型文档手动导出，再小批量重试 |
| Chrome 未登记下载任务 | 页面导出没有产生浏览器下载 | 重新加载扩展；MindNotes 应使用 DOM 生成模式，不依赖页面菜单 |
| FreeMind XML 无效 | 节点树未完整加载或 DOM 结构变化 | 打开脑图等待完整渲染后重试，并检查页面选择器 |
| 运行按钮禁用 | 尚未选择有效清单或没有该格式的可处理任务 | 检查清单列名、任务类型和当前断点状态 |

## 全量前检查表

- Chrome 已登录正确租户，源链接可以打开；
- 扩展是本仓库最新代码，并已在 `chrome://extensions/` 重新加载；
- 清单、下载目录和归档目录均不在 Git 仓库内；
- Chrome 下载目录空间充足；
- 小样中的 Word、Markdown 附件、Excel、PowerPoint 和 FreeMind 均能正常打开；
- 并发数和限速保持默认，除非已通过小批量验证；
- 最终归档输出目录是新的空目录。

## 发布日志前脱敏

日志和清单常含敏感信息。公开问题前删除或替换：

- 文档标题和文件夹名称；
- 完整 URL、document token 和租户子域名；
- 本地用户名及绝对路径；
- 组织名称、账号信息和内部错误上下文。

保留文档类型、HTTP 状态、飞书错误码、扩展版本和经过匿名化的最小复现步骤即可。
